1//! Common pinout
2//!
3//! The Teensy boards share many pins. This module provides
4//! the pins that are common across both boards. For pins that are unique to
5//! each board, and to acquire all of a board's pins, see
6//!
7//! - [`t40`](super::t40)
8//! - [`t41`](super::t41)
9//! - [`tmm`](super::tmm)
10//!
11//! ## Common pin table
12//!
13//! This table describes all common pins and their possible functions. Besides this table,
14//! there are two other ways to identify which pads support which peripheral:
15//!
16//! - study the i.MX RT 1060 Reference Manual. This is the authority on pad configuration.
17//! - study the trait implementations for the pad. Select a pin type alias, like [`P0`],
18//!   and click-through to its pad documentation (`AD_B0_03`). Notice the listing of `imxrt-iomuxc`
19//!   trait implementations. This describes what kinds of functions the pin supports. The constraints
20//!   may be enforced by the HAL's APIs.
